Grid

The grid is a responsive percentage based grid that uses the same markup and class names as the bootstrap 2.0 grid. This is intentional to support oscar projects if required. The grid is generated in the file grid.less and contains several variables to easily reconfigure the grid if required, e.g. to have a different number of columns.

It collapses to a single column below a certain screen width. The first element in a row automatically has no left margin. See the grid test file for a full grid layout.

Grid within grid, fixed spacing

The grid is based on percentages, including the space between grid columns. This means that a grid row within a grid row will have a narrower gap between columns because the same percentage of a narrower column is smaller (see above example).

For grids that must maintain a fixed column spacing, the .fixedspacing class can be applied to the parent grid row. This will ensure that the spacing between columns remains consistent even for nested grids.

This approach is considerably more limiting as the fixed column spacing must be split between the left and right of each column, meaning that the first left column will not align perfectly left as in the regular grid. e.g. if the column spacing is 20px, each column receives 10px of padding to the left and right.

Grid indents can also be applied to the fixed spaced grid as normal.

Reverse layout

When collapsed into a single column the grid translates left to right into top to bottom, meaning that the first column in a row will appear above the second column, and so forth.

For some layouts, this approach is not suited. Where this approach should be reversed, apply the .reverselayout class to the parent row, and reverse the order of the elements in the page, as shown.

Reverse layout can also be applied to the fixed spaced grid.

This column appears on the right at full size, but first when collapsed.

This column occurs in the middle.

This column appears on the left at full size, but last when collapsed.

Code


<div class="row reverselayout">
    <div class="span4">
        <p>This column appears on the right at full size, but first when collapsed.</p>
    </div>
    <div class="span4">
        <p>This column occurs in the middle.</p>
    </div>
    <div class="span4">
        <p>This column appears on the left at full size, but last when collapsed.</p>
    </div>
</div>

Non-collapsing grid

Occasionally it can be useful to have grid elements that do not collapse with the rest of the grid on a narrow screen. To achieve this add the class of .fix to the row element you do not wish to collapse.

Buttons

Buttons are controlled through the .btn class, with additional classes to control the appearance and size.

Plugins and JavaScript

Class naming and use conventions

All code on the site should adhere to the following convention for class naming and use with regard to JavaScript.

  1. IDs should only be used for JavaScript, not for styling.
  2. Any class used for JavaScript should be prefixed with "js-" and not have any styles associated with it.
  3. JavaScript can also be activated through the use of the data attribute on elements.

Note that the JS included with the house styles is meant as a starting point only. In its current form, the JS is in places inefficient or even inappropriate depending on how it will be used, and you are advised to tailor it specifically to your needs.

Popups

Popups are hidden elements at the foot of the page revealed by a click action on a specific element. By default popups are positioned 10% from the top of the current window position, unless the data-position="fixed" attribute is added to the link.

  • Standard popup (large version)
  • Fixed position (large version)
